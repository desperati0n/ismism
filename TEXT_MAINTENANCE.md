# 文字内容维护指南（主义主义）

本项目的文字分为两类：
- 数据文案（主义数据库）
- 界面文案（按钮/提示/标题）

本文说明各文案所在位置、修改方式以及新增/删除的注意事项。

---

## 一、数据文案（可扩展的内容数据）
位置：`data/isms.ts`

- 类型：`Ism { code: string; name: string; description: string }`
- 数据源：`ismsDatabase: Ism[]`
- 搜索逻辑：`searchIsms(searchCode: string)`

### 1) 如何新增一个主义
1. 打开 `data/isms.ts`
2. 在 `ismsDatabase` 数组中，新增一个对象：
   ```ts
   {
     code: '1-2-3-4',
     name: '你的主义名称',
     description: '对该主义的简短说明（可 1-3 段）',
   }
   ```
3. 保存后，运行页面，使用四格选择对应编码进行搜索即可看到新条目。

注意事项：
- `code` 必须是 4 段，用 `-` 分隔；每段取值为 `'1' | '2' | '3' | '4' | '$'`。
- `$` 的含义：
  - 搜索时：
    - 第一格的 `$` 表示“主体性缺失”，只匹配数据库中 `code` 以 `$-` 开头的条目。
    - 其他位置的 `$` 表示通配符（匹配任意 1-4）。
  - 数据库中实际条目通常只在“第一位”可能使用 `$`（表示特殊主题）。非首位的 `$` 尽量不要出现在数据库里（避免语义混乱）。
- 名称和描述支持中文，建议保持风格一致、语义清晰。

### 2) 如何修改或删除条目
- 修改：在 `ismsDatabase` 内找到对应 `code`，直接编辑 `name`/`description`。
- 删除：直接删除该对象；若被搜索依赖，页面只是不再展示该结果，不会报错。

## 二、界面文案（UI 文案）
界面固定文案分布在以下文件中：

### 1) `App.tsx`
- 顶部导航中的当前搜索展示：`搜索: {currentSearch}`
- 搜索结果数：`找到 {searchResults.length} 个匹配的主义`
- 空结果提示：`未找到匹配的主义`、`尝试调整格子中的值或使用通配符 $ 来扩大搜索范围`
- 首次进入引导文案：
  - `滑动格子选择元素，然后点击搜索按钮`
  - `• 数字 1, 2, 3, 4 代表不同的哲学维度`
  - `• 第一个格子的 $ 代表"主体性缺失"`
  - `• 其他格子的 $ 作为通配符使用`
- 详情页滚动行为（无用户可见文案，但属于交互逻辑）：
  - 返回列表时：立即跳回原滚动位置
  - 进入详情时：平滑滚动到顶部

维护方式：直接在文件内搜索相应中文文本并编辑。

### 2) `components/TopNavigation.tsx`
- 标题：`主义主义`
- 标签：`搜索:`
- 按钮：`重置`

### 3) `components/SearchButton.tsx`
- ARIA 标签：`aria-label="Search"`（若需中文可改为 `"搜索"`）

### 4) `components/ResultCard.tsx`
- 点击提示（在可点击时显示）：`点击查看详情 →`

### 5) `components/IsmDetail.tsx`
- 返回按钮：`返回搜索结果`
- 编码标签：`编码：`
- 小标题：`详细说明`、`哲学维度解析`
- 维度标签：`第一维度 / 第二维度 / 第三维度 / 第四维度`
- 含 `$` 的提示：
  - 首位 `$`：`主体性缺失（$）`
  - 其他位 `$`：`通配符（$）`

### 6) `components/GridSlider.tsx`
- 箭头按钮的 ARIA 标签：`Previous` / `Next`（若需中文可改为 `上一项` / `下一项`）
- 元素取值显示：`'1' | '2' | '3' | '4' | '$'`

## 三、批量替换与多语言建议（可选）
- 批量替换：在编辑器中全局搜索中文片段（如“重置”、“未找到匹配的主义”等），即可快速定位。
- 多语言（未来可选）：
  - 将界面文案集中到 `i18n/zh.ts` 等文件中，页面使用键值引用，便于统一管理与切换语言。
  - 数据文案（`ismsDatabase`）保持中文或多语言字段（如 `nameEn`/`descriptionEn`），按需渲染。

---

## 四、改动流程建议
1. 新增/修改 `ismsDatabase` 条目
2. 根据需要调整界面文案
3. 本地运行 `npm run dev` 自测
4. 提交到 Git：
   ```bash
   git add .
   git commit -m "docs: update texts and isms content"
   git push
   ```

如需我把界面文案抽离成集中管理（i18n）或把数据库迁移到 JSON 文件，告诉我你的偏好即可。
